# Contributing Guide

Thank you for your interest in contributing to Tiny Windows Shutdown Scheduler!

## How to Contribute

### Report a Bug
1. Check the [Issues](https://github.com/pavanbadempet/Tiny-Windows-Shutdown-Scheduler/issues) to see if it's already reported
2. Create a new issue with:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - Your Windows version and .NET Framework version

### Suggest a Feature
1. Visit [Discussions](https://github.com/pavanbadempet/Tiny-Windows-Shutdown-Scheduler/discussions)
2. Share your idea and get feedback from the community
3. Once discussed, you can create an issue to track it

### Submit Code Changes
1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Build and test: `build.bat`
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

## Development Setup

### Requirements
- Windows 7 or later
- .NET Framework Developer Pack
- A text editor or Visual Studio

### Building
```powershell
build.bat
```

### Testing
1. Build the project
2. Run the generated EXE from `dist/`
3. Test all timer functions locally

## Code Style

- Keep code simple and readable
- Follow standard C# conventions
- Comment complex logic
- Keep the application lightweight
